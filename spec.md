# Spec: 修复 PDF 文件上传到 Supabase Storage 问题

## 问题描述

用户在前端页面上传 PDF 文件后，日志显示上传成功，但在 Supabase Storage Dashboard 中看不到上传的文件。

## 当前状态

根据日志分析：
- ✅ PDF 文本提取成功
- ✅ 上传 API 调用成功（返回 200）
- ✅ Public URL 生成成功
- ✅ 推送到飞书成功

但用户反馈在 Supabase Dashboard 中看不到文件。

## 可能的原因

1. **文件名编码问题**：中文字符被替换为下划线，导致难以识别
2. **客户名称为 "unknown"**：前端没有正确传递客户名称
3. **Bucket 权限问题**：可能使用了错误的密钥
4. **Dashboard 缓存/刷新问题**：需要手动刷新

## 解决方案

### 方案 1：改进文件名处理
- 保留部分可读的英文字符
- 添加原始文件名的映射记录

### 方案 2：添加上传确认机制
- 上传后立即查询验证
- 在控制台显示上传结果

### 方案 3：优化客户名称提取
- 在上传时就使用 AI 提取的客户名称
- 如果无法提取，使用文件名作为临时客户名称

## 实施步骤

1. 检查当前上传逻辑
2. 验证 Service Role Key 是否正确
3. 改进文件名处理逻辑
4. 添加上传验证步骤
5. 测试完整流程

## 验收标准

- [ ] 前端上传 PDF 后，可以在 Supabase Storage Dashboard 中看到文件
- [ ] 文件名清晰可读
- [ ] Public URL 可以正常访问
- [ ] 推送到飞书时 URL 正确
